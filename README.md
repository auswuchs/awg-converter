# AWG Converter

Конвертер конфигов **AmneziaWG**: переводит `.conf` файлы в `vpn://` ключи и обратно. Работает полностью в браузере — ключи и приватная информация никуда не отправляются.

## Зачем это нужно

В AmneziaVPN можно поделиться сервером тремя способами: QR-код, файл `.conf` и ссылка `vpn://`. Последний вариант — самый удобный для пересылки в мессенджерах, но получить такую ссылку из существующего `.conf` файла из коробки нельзя. Этот конвертер закрывает дырку:

- У тебя есть `.conf` файл WireGuard/AmneziaWG → получи `vpn://` ссылку для шеринга
- Тебе прислали `vpn://`, но нужен `.conf` (например для роутера или другого клиента) → раскодируй обратно

## Фичи

- 🔒 **Всё локально** — никакого бэкенда, ключи не покидают браузер
- 🔄 **В обе стороны** — `.conf → vpn://` и `vpn:// → .conf`
- 🛡️ **AmneziaWG 2.0** — поддержка всех параметров обфускации (`H1-H4`, `S1-S4`, `Jc/Jmin/Jmax`, `I1-I5`)
- 📋 **Импорт в Amnezia** — генерируемые ссылки совместимы с официальным приложением AmneziaVPN
- ⚡ **Без зависимостей** — встроенные браузерные API для сжатия и base64

## Как пользоваться

1. Открой приложение
2. Выбери направление — `.conf → vpn://` или `vpn:// → .conf`
3. Вставь конфиг в текстовое поле
4. Жми "Конвертировать"
5. Копируй результат

## Как это работает (технически)

Формат `vpn://` в Amnezia — это:

```
vpn:// + URL-safe base64( qCompress( JSON ) )
```

где `qCompress` — Qt-шный формат сжатия: 4 байта (big-endian) размера несжатых данных + zlib-сжатые данные. JSON внутри имеет специфическую структуру с двумя уровнями вложенности — внешний `containers[0].awg` и внутренний `last_config` (JSON в виде строки), которые приложение парсит для добавления сервера.

Конвертер реализует обе стороны: парсит `.conf`, собирает правильную JSON-структуру, сжимает через `CompressionStream` API (с ручной коррекцией adler32, потому что браузеры считают его неправильно), и кодирует в URL-safe base64.
